How the Model Works

Decision tree classifier on stellar properties
The core of this project is a Decision Tree classifier, a supervised machine learning algorithm that learns a set of if‑then rules from labeled examples. Each star in the training dataset is described by four physical features: temperature, luminosity, radius, and absolute magnitude. The decision tree repeatedly splits the data based on thresholds on these features, such as “temperature > 10,000 K” or “luminosity < 0.01 L/Lo”, to separate different star types into purer groups. At the end of training, each leaf of the tree corresponds to a region of the feature space where most stars belong to a single class, for example white dwarfs or supergiants. [web:73][web:80]
For a new star, the model follows exactly the same set of rules: it starts at the root of the tree and moves down, evaluating each condition on the input features until it reaches a leaf. The class with the highest proportion of training examples in that leaf becomes the prediction, and the relative frequency of each class in that leaf is used to compute a probability distribution over all six types, which appears in the interface as the “confidence” of the prediction. This approach works well on structured scientific data, where meaningful physical thresholds exist, such as the temperature ranges that separate hot blue stars from cooler red stars or the luminosity gaps between dwarfs and supergiants. [web:73][web:72]
The features in the CSV file have clear astrophysical interpretations. The temperature (in Kelvin) controls the color of the star and reflects the average surface energy of the emitted photons. The luminosity (in units of the Sun’s luminosity L/Lo) measures the total power output, while the radius (in units of the Sun’s radius R/Ro) tells how large the star is compared to the Sun. The absolute magnitude is a logarithmic measure of intrinsic brightness: lower or more negative magnitudes correspond to more luminous stars. Together, these quantities are strongly linked by the Stefan–Boltzmann law, which relates luminosity to temperature and radius, and they define a star’s position on the Hertzsprung–Russell diagram, a fundamental tool for classifying stars. [web:82][web:85]
During evaluation, the model is tested on a held‑out subset of stars that were not used for training. The metrics reported—accuracy, precision, recall, and F1‑score—summarize how well the decision tree distinguishes each class. Accuracy measures the overall fraction of correct predictions, while precision and recall focus on how reliably the model identifies a specific type, for example how often objects predicted as “white dwarfs” are truly white dwarfs and how many actual white dwarfs are recovered. The F1‑score is the harmonic mean of precision and recall. In this project, the model achieves very high scores on all classes, which reflects the fact that the star types in the dataset occupy well separated regions in temperature–luminosity space.
